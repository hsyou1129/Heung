# 로그인 페이지

## 1. 카카오 로그인

    기본 흐름:
    1. 사용자가 어플에 접속한다.
    2. 카카오 계정을 통해 로그인을 한다.

    예외 흐름:
    1a. 만약 처음 접속한다면 로그인 버튼이 뜬다.
    2a. 만약 로그인을 한적이 있다면 어플이 바로 실행된다.

# 메인 페이지

## 1. 전체 게시글

    기본 흐름:
    1. 사용자가 로그인한다.
    2. 최신 게시글 순서로 게시글의 제목을 5개 노출한다.
    3. 사용자가 "더보기"를 클릭하여 전체 게시물 목록 페이지로 이동한다.
    
    예외 흐름:
    2a. 만약 게시글이 존재하지 않다면 공백으로 둔다.

## 2. 인기 게시글

    기본 흐름:
    1. 사용자가 로그인한다.
    2. 추천수를 기준으로 인기 게시글의 제목을 5개 노출한다.
    3. 인기 게시글 목록에서 하나를 클릭하면 게시글 내용 페이지로 이동한다.
    
    예외 흐름:
    2a. 만약 인기 게시글이 없다면 공백으로 둔다.

# 게시글 목록 페이지

## 1. 전체 게시글 목록

    기본 흐름:
    1. 사용자가 게시글 목록 페이지에서 전체 게시글 목록을 확인한다.
    2. 게시글 목록에는 게시글의 제목, 작성자 닉네임, 작성일, 추천수, 댓글수가 표시된다.
    3. 사용자가 게시글을 선택하면 해당 게시글 내용 페이지로 이동한다.
    
## 2. 인기 게시글 목록

    기본 흐름:
    1. 사용자가 인기 필터 버튼을 클릭하면 인기 게시글 목록을 확인할 수 있다.

## 3. 게시글 작성

    기본 흐름:
    1. 사용자가 게시글 목록 페이지에서 글쓰기 버튼을 클릭한다.
    2. 게시글 작성 페이지로 이동한다.

---

# 게시글 내용 페이지

## 1. 게시글 삭제

    기본 흐름:
    1. 사용자가 자신이 작성한 게시글을 클릭하면 게시글 내용 페이지로 이동한다.
    2. 사용자가 게시글 내용 페이지에서 삭제 버튼을 클릭한다.
    3. 게시글이 삭제되고, 게시글 목록에서도 삭제된다.

## 2. 게시글 수정

    기본 흐름:
    1. 게시글 작성자가 자신이 작성한 게시글을 클릭하면 게시글 내용 페이지로 이동한다.
    2. 게시글 작성자가 수정 버튼을 클릭한다.
    3. 게시글 작성 페이지로 이동한다.
    
    예외 흐름:
    3a. 게시글 작성 페이지에 본인이 작성한 내용이 작성되어 있다.

## 3. 게시글 추천

    기본 흐름:
    1. 사용자가 다른 사용자가 작성한 게시글을 클릭하면 게시글 내용 페이지로 이동한다.
    2. 사용자가 게시글 하단에 있는 추천 버튼을 클릭한다.
    3. 추천 수가 증가한다.
    
    예외 흐름:
    3a. 어플의 인기 게시글 기준을 넘으면 게시글은 인기 게시글 목록에 등록된다.

## 4. 추가 기능

    기본 흐름:
    1. 사용자는 댓글 작성을 할 수 있다.
    2. 사용자는 대댓글 작성을 할 수 있다.

# 게시글 작성 페이지

## 1. 게시글 작성

    기본 흐름:
    1. 사용자가 게시글 목록 페이지에서 게시글 작성 버튼을 클릭한다.
    2. 게시글 작성 페이지로 이동한다.
    3. 사용자가 제목과 내용을 입력한다.
    4. 사용자가 업로드 버튼을 클릭해 게시글을 작성한다.

## 2. 모집글 작성

# 모집글 작성 페이지

## 1. 모집글 작성

    기본 흐름:
    1. 사용자가 모집글 목록 게시판에서 글쓰기 버튼을 클릭한다.
    2. 모집글 작성 페이지로 이동한다.
    3. 사용자가 "버스킹"이나 "클래스"를 선택한다.
    4. 선택한 항목에 따라 사용자에게 각자 다른 사항을 요구한다.

    예외 흐름:
    4a. 버스킹일 경우 제목, 내용, 작성자 연락처, 모집 종료 일시, 세션 유형, 모집 최대 인원을 요구한다.
    4b. 클래스일 경우 제목, 내용, 작성자 연락처, 모집 종료 일시, 클래스 유형, 모집 최대 인원을 요구한다.

# 공연일정 확인 페이지

## 1. 공연일정 확인

    기본 흐름:
    1. 사용자가 공연일정이 있는 날짜를 클릭한다.
    2. 캘린더 하단에 공연일정 정보가 표시된다.
    3. 공연일정 정보를 클릭하면 공연일정 내용 페이지로 이동한다.

    예외 흐름:
    1a. 공연일정이 있는 날짜에는 표시가 되어있다.

## 2. 공연일정 작성

    기본 흐름:
    1. 사용자가 상단에 공연일정 작성 버튼을 클릭한다.
    2. 공연일정 작성 페이지로 이동한다.

# 공연일정 내용 페이지

## 1. 공연일정 내용

    2. 공연일정 내용을 확인한다.

    예외 흐름:
    2a. 이전에 작성한 공연정보가 공연정보 작성 페이지에 채워져있다.

## 3. 공연일정 삭제

# 대여점 검색 페이지

## 1. 지도 화면 표시

    기본 흐름:
    1. 해당 페이지에 구글 지도를 표시한다.

    예외 흐름:
    1a. 초기 화면은 현재 사용자의 위치이다.
    1b. 화면을 움직여도 현재위치 버튼을 클릭하면 지도 화면이 현재 위치로 바뀐다.

## 2. 대여점 검색

    기본 흐름:
    1. 사용자가 검색바를 통해 시흥시의 행정동을 검색한다.
    2. 검색한 행정동에 위치한 공연장, 대여점, 강의실이 지도에 표시된다.

    예외 흐름:
    2a. 지도 상단에 있는 스피너를 통해 지도에 표시할 가게 종류를 선택할 수 있다.

## 3. 대여점 정보 표시

    기본 흐름:
    1. 사용자가 검색 후 지도에 표시된 마커를 클릭하면 해당 가게의 정보를 표시한다.

    예외 흐름:
    1a. 가게의 정보에는 가게 이름, 주소, 가게 유형, 전화번호가 있다.

# 프로필 페이지

## 1. 작성글 확인

    기본 흐름:
    1. 사용자는 본인이 작성한 게시글 목록을 확인한다.
    2. 목록에서 게시글을 클릭하면 게시글 페이지로 이동한다.

## 2. 프로필 사진 변경

    기본 흐름:
    1. 사용자는 프로필 사진 변경 버튼을 클릭한다.
    2. 사용자의 갤러리에서 원하는 사진을 선택하여 프로필 사진으로 변경한다.

## 3. 닉네임 변경

    기본 흐름:
    1. 사용자가 프로필 하단에 있는 닉네임 변경 버튼을 클릭한다.
    2. 닉네임을 수정할 수 있는 팝업이 나타난다.
    3. 사용자가 새로운 닉네임을 입력하고 확인을 한다.
    
    예외 흐름:
    3a. 닉네임이 중복이라면 변경할 수 없다는 알람을 띄운다.

## 4. 설정

    기본 흐름:
    1. 버튼을 골라 다크모드를 설정한다.
    2. 버튼을 골라 언어를 선택한다.
    3. 버튼을 골라 로그아웃/회원탈퇴를 선택한다.

    예외 흐름:
    1a. 다크모드로 바꾸거나 시스템의 기본값으로 바꿀 수 있다.
    2a. 한국어나 영어 중 하나를 고를 수 있다.
